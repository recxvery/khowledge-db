# SMACSS*

В целом весь подход в том чтобы мы делили CSS на части. похуй как, главное чтобы делили

В отличии тут уже какой-то полный пиздец творится на фазе создания CSS. 1 файл CSS на целый блок?) ХАХАХАХАХА. НА НАХУЙ ДЕРЖИ НА ОДИН БЛОК

![image.png](2024-2025/Frontend%20development/CSS/Методологии%20и%20практики/SMACSS/image.png)

В basic - базовые правила. там условно, что в body, одинаковое поведение браузеров

в layout - гриды, флексы и пр. ГЛОБАЛЬНОЕ РАСПОЛОЖЕНИЕ ЭЛЕМЕНТОВ в том числе модулей. (l-row)

в module - расположение в блоке. (m-button)

state - для состояния. допустим hover и пр. и это должно быть в этот элемент (s-chipActive)

theme - для покрасок и внешнего вида (t-buttonPrimary)
