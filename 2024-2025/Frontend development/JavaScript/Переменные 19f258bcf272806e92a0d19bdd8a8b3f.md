# Переменные

# var / let / const

## var

не используется так как она не ограничивается блочными элементами(лишь функциями) т.е. спокойно поднимается наверх.

```jsx
var i = 321;

function donthg() {}; 

  for (var i = 0; i < 10; i++) {
        donthg();
  }
console.log(i) //10
```

```jsx
var i = 321;
var i = 254;    
```

И эта технология ещё и предусматривает hoisting, то есть выбрасывание на самый самый верх 

```jsx
console.log(s) //undefined

var s = 10
```

```jsx
var a = 10;

console.log(a + s); //undefined

var s = 10;
//NaN
```

Короче как правильно подметили:

> **var - как езда на мотоцикле, вроде бы и так нормально, но может и не повезти**
> 

## let / const

- ограничены блочной областью видимости и функциональной
- let i = 321;
let i = 254;    //нельзя переиницилазировать переменную.
- Видны только после объявления. и не инициализируется сразу же после объявления как в var.

## let vs const(diffs)

- у переменной заданной let можно изменить значение, а у const нет. Тем не менее объект определённый с помощью const. может быть изменён

```
const language = {
  name: 'JS'
}

language.name = 'Python'

console.log(language)//{ name: 'Python' }
```

- **const >>> let** т.к. добавляют коду безопасности и улучшает читаемость
